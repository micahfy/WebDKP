## 多团支持功能 — 完整实施方案

### 准则
- **WebDKP.lua 格式绝对兼容**：`WebDKP_Tables` 用 `{id, name}` 结构，`WebDKP_DkpTable` 用 `dkp_<tableid>` 字段
- **插件内团队隔离**：计分/导入/导出/备份/替补队长 都只操作当前团

---

### 第一部分：修复字段名 bug（前提，必须先做）

把所有 `"dkp"..tableid`（缺下划线）改成 `"dkp_"..tableid`：
- `ADKP.lua:871`（ADKP_AddPlayer 新玩家创建）
- `ADKP.lua:2521`（玩家详情打印）
- `ADKP.lua:2653, 2655`（ADKP_SelectAll）
- `ADKP.lua:7843`
- `GroupFunctions.lua:163`（ADKP_UpdateTableToShow 初始化）

共 6 处，全部改成带下划线的标准格式。

---

### 第二部分：团管理核心

**1. 默认团初始化**（`ADKP_ADDON_LOADED`，行1812 附近）
- 加载后若 `WebDKP_Tables` 为空（无任何团），自动创建默认团：
  ```lua
  WebDKP_Tables[1] = { id = 1, name = "DKP" }
  ADKP_Frame.selectedTableid = 1
  WebDKP_Options["SelectedTableId"] = 1
  ```
- 若已有团，验证 `selectedTableid` 指向的团还存在（防删除后残留）

**2. 新建团函数** `ADKP_CreateTable(name)`
- 遍历 `WebDKP_Tables` 找最大 id，新 id = 最大+1
- `WebDKP_Tables[newId] = { id = newId, name = name }`
- 自动切换到新团（调 `ADKP_SelectTable(newId)`）
- 刷新下拉框

**3. 删除团函数** `ADKP_DeleteTable(tableid)`
- 二次确认弹窗（StaticPopup）
- 物理清理 5 类数据：
  - `WebDKP_DkpTable[*]["dkp_<id>"]` 和 `["dkp<id>"]`（兼容 bug 变体）→ 遍历所有玩家删字段
  - `WebDKP_Log` 里 `entry.tableid == id` 的记录 → 先收集 key 再批量 nil（跳过 "Version"）
  - `WebDKP_Tables[id] = nil`
  - `WebDKP_Options["SubSettings"]["captains"][id] = nil`（第三部分改造后的结构）
  - 若删的是当前团 → 切到剩余的第一个团
- 刷新下拉框 + 主列表

**4. 切换团统一函数** `ADKP_SelectTable(tableid)`
- 改造 `ADKP_Tables_DropDown_OnClick`（行2734）提取成可复用函数
- 设置 `ADKP_Frame.selectedTableid` + `WebDKP_Options["SelectedTableId"]`
- 联动刷新：`ADKP_UpdateTableToShow()` + `ADKP_UpdateTable()` + `ADKP_Tab1_SyncChecks()`（刷新替补队长输入框）

**5. 下拉框 UI 改造**（`ADKP_Tables_DropDown`，行2688-2727）
- 显示规则改为：≥1 团就显示（原来是 ≥2），但单团时也显示让用户能新建第二个团
- 下拉菜单底部加两个特殊项：「+ 新建团...」「✕ 删除当前团」
- 新建团：弹输入框让用户输团名 → 调 `ADKP_CreateTable`
- 删除团：调 `ADKP_DeleteTable`（带二次确认）

### 第三部分：替补队长按团独立

**数据结构变更**：
```lua
-- 改前（全局）：
WebDKP_Options["SubSettings"] = { captain = "" }

-- 改后（按团）：
WebDKP_Options["SubSettings"] = { captains = { [1] = "玩家A", [2] = "玩家B" } }
```

**改造读写链路**（优先改枢纽函数，散点逐一处理）：
- `ADKP_ResolveSubCaptain`（行10623）：读改为 `SubSettings.captains[ADKP_GetTableid()]`，写同理
- `ADKP_Tab1_SaveSubCaptain`（行10111）：保存到 `captains[currentTableid]`
- `ADKP_Tab1_SyncChecks`（行10065）：从 `captains[currentTableid]` 恢复输入框
- `ADKP_InitSubSettings`（行986）：初始化时做格式迁移（旧 `captain` 字段 → `captains[1]`）
- `ADKP_Z_GetCaptainName`（行5226）：fallback 改读 `captains[tableid]`
- 散点（行10033, 6156, 6771, 9937 等）：统一走 `ADKP_ResolveSubCaptain` 或按 tableid 读取

### 第四部分：导入/导出/备份按团过滤

**1. 导入初始分**（`ADKP_DoImportInitial`，行10136）
- 改为只清当前团：
  - 不再 `WebDKP_DkpTable = {}`，改为遍历玩家删 `dkp_<currentTableid>` 字段
  - 不再 `WebDKP_Log = {}`，改为删 `tableid == currentTableid` 的记录
  - 导入的初始分写到 `dkp_<currentTableid>`
  - 新增日志记录带 `tableid = currentTableid`

**2. 导出当前记录**（`ADKP_BuildExportText` + `ADKP_BuildBackupCSV`）
- `ADKP_BuildExportText`：加 `entry.tableid == ADKP_GetTableid()` 过滤
- `ADKP_BuildBackupCSV`：同样加 tableid 过滤
- 备份文件名加 tableid 标识（可选，便于区分）
